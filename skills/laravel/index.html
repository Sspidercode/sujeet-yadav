1. Routing

Laravel routes map URLs to actions (closures, controllers). Supports HTTP methods (GET, POST, PUT, DELETE), route parameters, named routes, groups, and middleware. Defined in routes/web.php or routes/api.php.

// Basic route
Route::get('/users/{id}', function ($id) {
  return "User ID: $id";
});

// Named route with middleware
Route::get('/profile', [ProfileController::class, 'show'])->name('profile')->middleware('auth');

// Resource route (RESTful)
Route::resource('posts', PostController::class);

// Route group
Route::prefix('admin')->group(function () {
  Route::get('/dashboard', [AdminController::class, 'index']);
});
2. Controllers

Controllers handle route logic, separating concerns. Generated via php artisan make:controller. Support dependency injection, resource controllers, and method-specific requests.

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class UserController extends Controller
{
  public function index()
  {
      $users = User::all();
      return view('users.index', compact('users'));
  }

  public function store(Request $request)
  {
      $validated = $request->validate([
          'name' => 'required|string|max:255',
          'email' => 'required|email|unique:users',
      ]);

      User::create($validated);
      return redirect('/users');
  }
}
3. Eloquent ORM

Eloquent provides an active-record implementation for database interactions. Models extend Illuminate\Database\Eloquent\Model. Supports relationships (hasOne, hasMany, belongsTo, many-to-many), queries, and mutators.

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Post extends Model
{
  protected $fillable = ['title', 'content', 'user_id'];

  // Relationship
  public function user()
  {
      return $this->belongsTo(User::class);
  }

  // Query scope
  public function scopePublished($query)
  {
      return $query->where('published_at', '<=', now());
  }
}

// Usage
$posts = Post::published()->with('user')->get();
$latest = Post::latest()->first();
4. Migrations

Migrations are version-controlled database schema changes. Created with php artisan make:migration. Use up() for apply, down() for rollback. Supports schema building (tables, columns, indexes).

<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
  public function up()
  {
      Schema::create('posts', function (Blueprint $table) {
          $table->id();
          $table->string('title');
          $table->text('content');
          $table->foreignId('user_id')->constrained()->onDelete('cascade');
          $table->timestamps();
      });
  }

  public function down()
  {
      Schema::dropIfExists('posts');
  }
};

// Run: php artisan migrate
5. Blade Templates

Blade is Laravel's templating engine for views. Supports directives (@if, @foreach, @extends), components, sections, and escaping. Views are in resources/views.

<!-- resources/views/layouts/app.blade.php -->
<!DOCTYPE html>
<html>
<head>
  <title>@yield('title')</title>
</head>
<body>
  @yield('content')
</body>
</html>

<!-- resources/views/users/index.blade.php -->
@extends('layouts.app')

@section('title', 'Users')

@section('content')
  @foreach($users as $user)
      <h2>{{ $user->name }}</h2>
      @if($user->isAdmin)
          <p>Admin user</p>
      @endif
  @endforeach
@endsection
6. Middleware

Middleware filters HTTP requests (e.g., auth, rate limiting). Created with php artisan make:middleware. Registered in app/Http/Kernel.php or routes.

<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;

class CheckAge
{
  public function handle(Request $request, Closure $next)
  {
      if ($request->age <= 200) {
          return redirect('home');
      }

      return $next($request);
  }
}

// In routes/web.php
Route::get('/admin', [AdminController::class, 'index'])
  ->middleware(CheckAge::class);

// Global middleware in Kernel.php: protected $middleware = [...]
7. Authentication

Laravel Sanctum/Breeze/Jetstream for auth. Uses guards, providers. Commands like php artisan make:auth (older) or Breeze for scaffolding. Supports sessions, tokens.

// In controller
use Illuminate\Support\Facades\Auth;

public function login(Request $request)
{
  $credentials = $request->validate([
      'email' => 'required|email',
      'password' => 'required',
  ]);

  if (Auth::attempt($credentials)) {
      $request->session()->regenerate();
      return redirect()->intended('dashboard');
  }

  return back()->withErrors(['email' => 'Invalid credentials']);
}

// Logout
Auth::logout();

// Middleware: ->middleware('auth')

// Sanctum for API: composer require laravel/sanctum
8. Queues

Queues handle background jobs for scalability. Uses drivers (database, Redis). Jobs created with php artisan make:job. Dispatched via dispatch().

<?php

namespace App\Jobs;

use Illuminate\Bus\Queueable;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Foundation\Bus\Dispatchable;
use Illuminate\Queue\InteractsWithQueue;
use Illuminate\Queue\SerializesModels;

class SendEmail implements ShouldQueue
{
  use Dispatchable, InteractsWithQueue, Queueable, SerializesModels;

  public function handle()
  {
      // Send email logic
      Mail::to('user@example.com')->send(new WelcomeMail());
  }
}

// Dispatch
SendEmail::dispatch($user);

// Config in .env: QUEUE_CONNECTION=database
// Run: php artisan queue:work
9. Events and Listeners

Events decouple code; listeners react to them. Generated with php artisan make:event and make:listener. Registered in EventServiceProvider.

<?php

namespace App\Events;

use Illuminate\Foundation\Events\Dispatchable;
use Illuminate\Queue\SerializesModels;

class UserRegistered
{
  use Dispatchable, SerializesModels;

  public $user;

  public function __construct($user)
  {
      $this->user = $user;
  }
}

<?php

namespace App\Listeners;

use App\Events\UserRegistered;

class SendWelcomeEmail
{
  public function handle(UserRegistered $event)
  {
      // Send email to $event->user
  }
}

// In UserController
event(new UserRegistered($user));

// In EventServiceProvider
protected $listen = [
  UserRegistered::class => [SendWelcomeEmail::class],
];
10. Testing

Laravel uses PHPUnit/Dusk for unit, feature, browser tests. Assertions, factories, refreshDatabase trait. Run with php artisan test.

<?php

namespace Tests\Feature;

use Illuminate\Foundation\Testing\RefreshDatabase;
use Tests\TestCase;
use App\Models\User;

class UserTest extends TestCase
{
  use RefreshDatabase;

  public function test_user_can_login()
  {
      $user = User::factory()->create();

      $response = $this->post('/login', [
          'email' => $user->email,
          'password' => 'password',
      ]);

      $response->assertRedirect('/dashboard');
      $this->assertAuthenticated();
  }
}

// php artisan make:test UserTest --unit
// php artisan test
11. Policies

Policies authorize actions on models. Created with php artisan make:policy. Used with authorize() or gates in views.

<?php

namespace App\Policies;

use App\Models\User;
use App\Models\Post;

class PostPolicy
{
  public function update(User $user, Post $post)
  {
      return $user->id === $post->user_id;
  }
}

// In Controller
$this->authorize('update', $post);

// In Blade: @can('update', $post) ... @endcan

// Register in AuthServiceProvider
protected $policies = [
  Post::class => PostPolicy::class,
];
12. Deployments

Deploy via Forge/Envoyer, or manual (Composer install, env config, migrate). Use php artisan optimize for production. CI/CD with GitHub Actions.

// .env production
APP_ENV=production
APP_DEBUG=false
DB_CONNECTION=mysql
DB_HOST=...

// Deployment script (e.g., in deploy.php)
server('production', 'server.example.com')
  ->stage('production')
  ->user('deployer');

task('deploy', [
  'deploy:prepare',
  'deploy:vendors',
  'deploy:release',
  'deploy:migrate',
  'deploy:optimize',
  'deploy:publish',
  'deploy:symlink',
  'deploy:restart',
  'cleanup',
]);

This covers core Laravel concepts. For advanced topics like Horizon (queues), Nova (admin), or API resources, refer to official docs. Always follow Laravel's coding standards for maintainability.
